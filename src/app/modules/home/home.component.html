Signals in Angular v16

Signals provide a new way for our code to tell our templates (and other code) that our data has changed. This improves Angular's change detection, which also improves performance, and makes our code more reactive.

A signal is a special type of variable that holds a value. But unlike other variables, a signal also provides notification when the variable value changes.

The signal() method must always be supplied with a value. If no data type is specified, the method reads the type from the given value in the function parameter.

There are two types of signals:

  • Signal
  • WritableSignal

Methods for Signal

  • [set(value: T): void] - set the signal to a new value, and notify
  • [update(updateFn: (value: T) => T)] - update the value of the signal based on its current value, and notify
  • [mutate(mutatorFn: (value: T) => void) - update the current value by mutating it in-place, and notify

RxJS Interop

  • fromObservable() - function converts a observable into an signal
  • fromSignal() - function converts a signal into an observable

Both methods are from the package '@angular/core/rxjs-interop'
